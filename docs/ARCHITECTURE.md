# Architecture

## Overview
Lotto Activo is a React + Vite platform with a feature-driven structure.
Main goals:
- Fast loading in Vercel with code splitting.
- Mock APIs per game while production APIs are pending.
- Simulated real-time winner flow using a WebSocket-like service.

## Core Stack
- React 19
- Vite 5
- Tailwind CSS
- Zustand
- Framer Motion

## Modules
- src/components/ui: reusable atomic UI blocks.
- src/store: global state with Zustand.
- src/features/games: game data, game grid, game sync, sidebar, top results bar.
- src/features/streaming: stream player and winner roulette overlay.

## Data Flow
1. Sidebar selects active game.
2. useGameData fetches game payload from endpoint configured in env.
3. useGameSync listens to mock websocket winner events.
4. Top results and winner overlay update with latest winner.
5. Grid highlights current winning animal automatically.

## API Configuration
Environment variables are read in src/features/games/services/mockData.js.
If VITE_API_BASE_URL is empty, relative paths are used.
If VITE_API_BASE_URL is set, game paths are joined to that base URL.

## Current Game Endpoints
- VITE_API_GAME_LOTTO_ACTIVO
- VITE_API_GAME_LOTTO_INTER
- VITE_API_GAME_MONJE_MILLONARIO

## Prepared Product Endpoints
- VITE_API_PRODUCTS_LIST
- VITE_API_PRODUCTS_CATALOG
- VITE_API_PRODUCTS_PRICING
- VITE_API_PRODUCTS_AVAILABILITY

## Real-time Behavior
Winner events are generated by mockWebSocket service.
Default cycle is 90 seconds between winner events after data is available.
If data is not loaded yet, service retries quickly to avoid silent periods.
